## Working thesis

After the destroyed-structure finding, the defensible claim is a decomposition,
not a single emergence story:

1. **Geographic clustering of node embeddings** in the trained model is real,
   but it is *not* specifically sequence-learned — any training signal with
   geographic co-occurrence (including a bag-of-tokens / within-route-shuffled
   objective) produces it. It is approximately what node2vec produces from the
   graph alone.

2. **Graph-adjacency-using behavior** — the model's ability to assign high
   probability to a node's real graph neighbors — IS specifically
   sequence-learned. Destroyed-structure-trained models do not exhibit it.

3. The standard "an emergent metric map" framing conflates these. The
   methodology of probe-based controls in graph-data interpretability needs
   stricter destruction (set-membership-breaking, not just order-breaking) to
   distinguish them.

To support this honestly, three core experiments are mandatory.

## Must-do experiments

### Experiment 1 — Activation transplant for Phase 5

The current `eval/causal.py` uses pseudoinverse-derived patches that operate on
the probe's geometry. That doesn't isolate "model uses graph adjacency." Replace
it with model-own activations:

- Pick test position with current node A in some prefix.
- Pick a different "source" position (same or different sequence) where current
  node is B. Cache that position's residual at the chosen layer L: `a_B`.
- At the test position, *replace* the residual at layer L with `a_B`. Continue
  forward; measure next-token distribution.
- Score:
  - P(A's neighbors) — should fall toward 0
  - P(B's neighbors) — should rise toward `a_B`'s native level

**Falsifiable predictions:**

| Model | Predicted effect |
|---|---|
| Real-trained London | P(B's neighbors) jumps from ~0 → substantial (>0.3); P(A's neighbors) crashes |
| Destroyed-structure London | No systematic shift toward B's neighbors (model has no graph-adjacency knowledge to swap) |

**Effort:** ~half day of code (~250 lines, can reuse most of `eval/causal.py`'s
infrastructure). ~10 min runtime per city on M1.

**Deliverable:** New file or function in `eval/causal.py` for activation-
transplant intervention. Outputs a clean differential table comparing real and
destroyed-structure models.

### Experiment 2 — Global-shuffle destroyed-structure control

The current `--shuffle_routes` flag preserves route SET-MEMBERSHIP. Add a
stricter shuffle that breaks even that:

- Take `train.bin` after generation; randomly permute every real-token position
  across the entire stream (preserving BOS/EOS structure).
- Each "route" now contains random nodes drawn from the whole vocab.
- Retrain the same `small.py` model on this. Re-run `eval/probe.py`.

**Falsifiable predictions:**

| Condition | Predicted probe R² (node-level linear) |
|---|---|
| Real London | ~0.64 (already measured) |
| Within-route shuffle | ~0.96 (already measured — clustering survives) |
| **Global shuffle** | **near 0** (clustering should finally fail because there's no geographic co-occurrence anymore) |

If global shuffle ALSO produces non-zero R², the methodological story tightens:
even pure unigram co-occurrence is enough. If it's near zero, we have a clean
gradient.

**Effort:** ~30 min code (add `--shuffle_globally` to `prepare_city.py`) + ~30
min retrain + ~5 min probe.

**Deliverable:** New flag in `data/prepare_city.py`, retrained checkpoint,
probe result added to the cross-condition table.

### Experiment 3 — wte vs node2vec comparison

The single most informative experiment of the three for the framing question
"is the transformer doing anything more than co-occurrence learning?":

- Run node2vec on the City of London graph directly (`gensim` Word2Vec on
  random walks generated by node2vec sampling). Match `n_embd=384`. No
  transformer involved.
- Compare the resulting 384-dim node embedding matrix to the trained model's
  `wte` matrix (taking the 663 real-node rows).
- Procrustes alignment + R² of aligned residuals. CKA (centered kernel
  alignment) is also reasonable.

**Falsifiable predictions:**

| Outcome | Interpretation |
|---|---|
| Procrustes-CKA ≥ 0.9 between `wte` and node2vec | Trained model's input embedding is essentially node2vec. The transformer's contribution lives in higher layers. |
| 0.5 ≤ similarity < 0.9 | `wte` is partly node2vec-like with some additional structure. |
| similarity < 0.5 | `wte` is qualitatively different. Worth investigating what's different. |

**Effort:** ~3 hours of code. No GPU; node2vec runs on CPU. Pure analysis
script; doesn't modify any other code.

**Deliverable:** `eval/embedding_compare.py` (new file). Outputs per-city
similarity numbers and per-layer probe comparison (does probing node2vec
embeddings directly give the same R² as probing `wte`?).

## Strongly-should-do experiments

These don't change the core thesis but materially strengthen it.

### Experiment 4 — LSTM baseline

PLAN.md Phase 3 deferred item. Train a same-parameter-count LSTM on real
London with the same data + protocol, then run every eval we ran on the
transformer.

**Why it matters:** if LSTM also shows the clustering-vs-adjacency
decomposition, the finding is architecture-general (sequence-model-general).
If only the transformer shows it, the finding is transformer-specific. Either
answer is informative.

**Effort:** ~half day to write the LSTM class + adapt train loop; ~30 min
training on M1; ~10 min for the full eval suite.

**Deliverable:** `model/lstm.py`, `model/configs/lstm_small.py`, eval results
for LSTM on London.

### Experiment 5 — Cross-city probe transfer

Train probe on Manhattan model's L3 activations. Apply to Boston model's L3
activations. Does it generalize?

**Why it matters:** if the geographic encoding transfers, there's a "general
basis" for residual-stream geography across models. If not, each model's
encoding is idiosyncratic and what we found is per-model.

**Effort:** ~1 hour using existing artifacts. No training.

**Deliverable:** small additional script or flag on `eval/probe.py` for
cross-checkpoint probe transfer. Result table.

Medium config, full corpus, 30k iters on a rented 4090 (~$2, ~2 hours).

### Experiment 7 — `viz/overlay.py`

Procrustes-aligned recovered-vs-true overlay. Best done after Experiment 6
provides tighter coordinates. ~half day to implement; cleanest output requires
tight probe error first.

### Experiment 8 — Geographic-region holdout

`--split geographic` in `prepare_city.py`. Hold out a contiguous lat/lon
sub-region. Especially relevant post-destroyed-structure finding: region
holdout breaks set-membership co-occurrence for held-out nodes in a different
way than node-level random holdout.

**Effort:** ~half day code + corpus regen + retrain. Skippable for the core
thesis but interesting as a generalization test.

## What we are NOT doing (explicitly out of scope)

- Strengthening the "linearly encoded" claim. The MLP-probe contamination on
  continuous targets is a real obstacle; we can't fix it without a different
  test methodology, and that's a separate research question.
- Claiming a metric-map result without the activation-transplant test working.
  The probe-decoded clustering is real but not by itself a strong enough
  finding to carry the project.
- Scaling up to multi-million-node graphs. The current 4-city setup is enough.

## Open methodological questions worth flagging for any future write-up

These are real open questions raised by today's findings:

- **Is "node-level split" enough as a probe-capacity control?** It's better
  than position-level, but the shuffled-trained model still passes it (R²
  0.97 on node-level). This means "node-level split" is *necessary but not
  sufficient* — passing it does not establish that the probe has found a
  sequence-specific representation.
- **What constitutes a fair "destroyed-structure" control for continuous
  targets on graph data?** Within-route shuffle is too weak; global shuffle
  may or may not be enough (Experiment 2 will tell). The general principle
  ("break the source of co-occurrence that drives the representation") is
  cleaner than the specific instantiation.
- **What is the relationship between graph-adjacency knowledge and the
  residual-stream representation?** If activation transplant works
  (Experiment 1), the residual encodes adjacency directly. If it doesn't,
  the model uses adjacency via a different mechanism (perhaps in the attention
  patterns, perhaps via the MLP). Either is interesting; both are accessible
  with our existing tooling.
