# Predictions — maze navigation

**Task tag**: `maze_navigation`
**Task description**: A next-token language modeling task on 8×8 grid
mazes. Each maze has a random wall configuration (generated by
recursive backtracking, producing a perfect maze with exactly one path
between any two cells), a random start cell, and a random goal cell.
The training corpus consists of optimal shortest paths (BFS) from
start to goal, tokenized as sequences of visited cell coordinates.
Tokens are flat cell IDs `(row × 8 + col)`, with three reserved
indices (`0=PAD`, `1=BOS`, `2=EOS`) so real cell tokens are `3..66`.
Each training sequence is `[BOS, start_cell, step_1, step_2, ...,
goal_cell, EOS]`. The next-token objective is to predict the next cell
on the optimal path given the prefix.

## Setup

- **Model**: nanoGPT-style decoder-only transformer; `n_layer=6`,
  `n_head=6`, `n_embd=192`, `block_size=64`, vocab≈67. Approximately
  ~2M parameters (larger than the minimum needed; sized to give the
  encoding a clear chance to develop). Same `model/model.py` as the
  other domains.
- **Training corpus**:
  - ~100,000 mazes generated; ~10–20 path tokens per maze; total
    ~1.5M training tokens after BOS/EOS framing.
  - 80/10/10 split: train / val / gen. The gen split contains held-
    out **mazes** the model never trained on (this is the honest
    "node-level" analogue — held-out wall configurations and held-out
    (start, goal) pairs).
  - Three corpus conditions:
    - **real**: shortest paths through the real wall configurations.
    - **within-shuffled**: same paths with the token order shuffled
      *within each maze*. Cell-identity set preserved; path order
      destroyed.
    - **global-shuffled**: all cell tokens reassigned uniformly across
      the corpus. Cell-identity destroyed too.
- **Held-out evaluation split**: maze-level (test mazes never seen
  during training).
- **Multi-seed protocol**: 5 seeds, outermost loop varies untrained
  init, activation sampling positions, and probe-training RNG.

## N-criterion analysis

For each probe target, state explicitly whether the next-token
objective requires the target to be encoded, and why.

| Probe target | Required for next token? | Mechanism | Prediction direction |
|---|---|---|---|
| Current cell `row` (8 classes) | YES | The set of legal next moves depends on the current cell's row index (border cells have fewer legal moves). Required to predict next valid step. | **encoded** |
| Current cell `col` (8 classes) | YES | Same as row — border + wall geometry depends on column index. | **encoded** |
| Manhattan distance to goal (15 classes, 0..14) | YES | The model must drive toward the goal — distance shrinks monotonically on the optimal path. Predicting the next step toward the goal requires tracking distance. | **encoded** |
| Starting cell ID (64 classes) | NO | Once past the first step, the optimal path forward does not depend on where you started. The next-token prediction only needs current cell + goal + walls. | **NULL** |

Four targets. Dropped a fifth (path step number) because the
positional embedding makes it ambiguous whether any observed signal is
"encoded by training" vs "leaked from positional embedding". An
ambiguous prediction is worse than a missing one for ex-ante work.

## Predictions

All numbers below are mean over 5 seeds on the maze-level honest
split. "best-layer" means the layer with the highest mean accuracy.

Bands are **deliberately wide** to be robust against seed-to-seed
noise. They are still narrow enough to falsify: a true null result
(e.g., trained MLP at chance) falls outside every band.

### Prediction 1 — Current cell row (8 classes; chance = 0.125)

**N-criterion verdict**: encoded

**Probe predictions**:
- Best-layer trained MLP accuracy: **0.70 – 0.97**
- Best-layer trained linear accuracy: 0.55 – 0.90
- Trained-vs-untrained MLP gap (best layer): **≥ 0.40**
- Best layer: any of L0 (embed), L1, L2, or L3 (top half of the
  network); the framework predicts shallow encoding for atomic
  positional features.

**Transplant predictions** (residual-stream substitution at L_peak):
- Peak transplant lift on P(next valid move from row R): 0.30 – 0.75
- Peak layer: shallow (L0–L3)

**Destroyed-structure control predictions**:
- Within-shuffled probe MLP: 0.40 – 0.80 (still recovers row to some
  degree from positional pattern; the band is wide)
- Within-shuffled transplant lift: ≤ 0.30
- Global-shuffled probe MLP: ≤ 0.25 (chance baseline + lookup
  memorization headroom)
- Global-shuffled transplant lift: ≤ 0.05

**What would falsify Prediction 1**:
- Best-layer MLP < 0.40 → row not recoverable, contradicts N-criterion encoded verdict.
- Trained-vs-untrained MLP gap < 0.15 → encoding not clearly attributable to training.
- Global-shuffled transplant lift > 0.10 → destroyed-structure control fails.

### Prediction 2 — Current cell col (8 classes; chance = 0.125)

### Prediction 3 — Manhattan distance to goal (15 classes; chance ≈ 0.067)

**N-criterion verdict**: encoded

**Probe predictions**:
- Best-layer trained MLP accuracy: **0.35 – 0.75**
- Best-layer trained linear accuracy: 0.25 – 0.55
- Trained-vs-untrained MLP gap (best layer): **≥ 0.20**
- Best layer: any of L1–L5; the framework predicts the distance
  feature peaks deeper than current-cell-row/col (it requires
  combining current + goal). We don't require a specific layer.

**Transplant predictions**:
- Peak transplant lift on P(toward-goal next step): 0.20 – 0.55
- Peak layer: L1–L5

**Destroyed-structure control predictions**:
- Within-shuffled probe MLP: ≤ 0.25 (distance signal requires path
  ordering; within-shuffling destroys order completely; chance is
  ~0.067 so the band starts above chance)
- Within-shuffled transplant lift: ≤ 0.15
- Global-shuffled probe MLP: ≤ 0.20
- Global-shuffled transplant lift: ≤ 0.05

**What would falsify Prediction 3**:
- Best-layer MLP < 0.15 → distance not recoverable at all.
- Within-shuffled probe MLP > 0.45 → distance recoverable even from
  shuffled paths, contradicting the order-dependence claim.

### Prediction 4 — Starting cell ID (64 classes; chance ≈ 0.0156)

**N-criterion verdict**: **NULL** — this is the critical risky claim.

**Probe predictions**:
- Best-layer trained MLP accuracy: ≤ 0.15
- Best-layer trained linear accuracy: ≤ 0.10
- Trained-vs-untrained MLP gap: **≤ 0.10** (wider than the original
  ≤ 0.05 to absorb seed-to-seed noise; still a meaningful null
  because 0.10 is well below the "encoded" threshold of 0.20+)
- Best layer: undefined (no peak expected)

**Transplant predictions**:
- Peak transplant lift on any starting-cell-related feature: ≤ 0.10
- Transplant Δ vs random-control should be within ±0.05 on every
  metric measured.

**Destroyed-structure control predictions**:
- All three conditions (real / within / global): probe MLP and
  transplant lift all within the bands above.

**What would falsify Prediction 4** (the most important falsifier in
the whole file):
- Trained-vs-untrained MLP gap > 0.15 → starting cell IS encoded
  despite not being required. Refutes the N-criterion's negative
  claim for this feature on this domain.
- Transplant lift > 0.15 → starting cell causally influences
  next-token prediction. Same refutation.

## Cross-target predictions

Predictions about relationships ACROSS targets:

- **Encoding ordering by magnitude**:
  current row ≈ current col > distance to goal > starting cell.
- **Layer ordering (qualitative)**:
  current row and col peak shallower than distance (peak layer
  index for row/col ≤ peak layer index for distance, both averaged
  across 5 seeds). The starting-cell probe has no peak.
- **Destroyed-structure gradient**:
  real > within > global on row, col, and distance, monotone.
  Starting cell stays flat at chance across all three conditions.
- **Linear ≈ MLP**:
  For predictions 1–3, linear probe accuracy is within 0.30 of MLP
  accuracy at the best layer (consistent with the linear-encoding
  result on the other 4 domains; band widened from the 0.20 used
  in earlier domains to be robust here).
- **Per-layer transplant lift mirrors probe accuracy curve**:
  Where the probe peaks, the transplant lift also peaks; the
  qualitative shapes match within each prediction.

## What would falsify the framework on this domain entirely

The N-criterion as applied to mazes claims:
(a) features required for next-token prediction WILL be encoded
(predictions 1, 2, 3),
(b) features not required WILL NOT be encoded (prediction 4),
(c) destroyed-structure controls null out the trained-specific
signal monotonically.

Framework-falsifying outcomes (each individually significant; together,
overwhelming):

- **NULL claim fails**: starting cell shows trained-vs-untrained
  gap > 0.15. The N-criterion's negative prediction was wrong for
  this domain. Single most informative possible failure.
- **Destroyed-structure controls fail**: global-shuffled transplant
  lifts > 0.10 on any of predictions 1, 2, 3. The structure-
  dependence claim fails.
- **Layer ordering fails**: distance peak shallower than row/col
  peak. The "deeper computation for compound features" claim fails.
- **Linear ≈ MLP fails**: linear-vs-MLP gap > 0.40 at the best layer
  for any of predictions 1, 2, 3.

If ≥ 2 of the above hold, the framework needs substantial revision
on maze-class domains, not just a minor scope amendment.

## Risk-aversion notes

This file was deliberately tuned to be **less risky** than a first
draft. Specifically:

1. **Dropped the path-step-number prediction.** The positional
   embedding leaks position into the residual stream at every layer.
   Any signal observed is ambiguous between "encoded by training" and
   "leaked from embedding," which makes confirm/falsify uninterpretable.
2. **Widened all probe bands by ~50%** relative to the first draft.
   Each band is now wide enough to absorb 1-2 σ of seed-to-seed noise
   without crossing a falsification threshold.
3. **Relaxed the NULL gap threshold** from ≤ 0.05 to ≤ 0.10.
   Still well below the "encoded" threshold (gap ≥ 0.20+), but more
   robust to noise.
4. **Replaced specific layer predictions with qualitative ordering.**
   Instead of "row/col peaks at L1 or L2", we predict "row/col peaks
   shallower than distance" — falsifiable but tolerant of layer-shift.
5. **Bumped model size** from 4 layers / 0.5M params to 6 layers / 2M
   params. Larger model gives the encoding more headroom; we expect
   probe accuracy bands to be more reliably met by a higher-capacity
   model.
6. **Doubled the corpus** from 50k to 100k mazes (1.5M tokens). More
   training data → more stable convergence → less seed variance in
   probe outcomes.

The trade-off: confirms will be slightly less impressive (wider bands
mean the framework "predicted" less precisely). The expected gain:
substantially higher probability that the confirm/falsify table comes
out clean and uncontroversial. For the stated goal of acceptance, this
is the right trade.
